0 · DERP embed là gì?

Đây không phải derper chính chủ của Tailscale — mà là DERP relay viết lại bằng Go, gọi là hybrid vì làm 2 việc cùng lúc:

PhíaCổngViệc làm
TCP — giao thức DERP v2:8080 (sau TLS Caddy/Traefik)Nhận kết nối DERP qua Upgrade: DERP, relay gói WireGuard
UDP — cầu WireGuard:41641/udpVới peer có UDP endpoint → gửi thẳng gói WireGuard qua UDP, không relay 2 chặng
Vì sao cần? Đúng kịch bản công ty chặn UDP (Squid): client TCP-only kết nối relay qua TCP/443 (được phép), rồi relay tự làm chặng UDP tới peer UDP-capable → client vẫn đạt hiệu năng gần direct thay vì double-relay.
Bảo mật: relay không đọc được nội dung. Payload là gói WireGuard mã hoá đầu-cuối (ChaCha20-Poly1305); relay chỉ thấy public key để định tuyến. Phần bắt tay dùng NaCl box (curve25519 + xsalsa20 + poly1305) để xác thực client sở hữu key — không mã hoá dữ liệu.

1 · Giao thức trao đổi (DERP protocol v2)

1.1 Khung tin (frame) — frame.go

type1 byte
length4 byte big-endian (uint32)
payloadlength byte

length tối đa maxFrameSize = 65535 + 64 (lớn hơn → lỗi “frame too large”). readFrame/writeFrame đọc/ghi đúng khuôn này.

2 · Call flow từng tính năng

F1 — Handshake / connect (server.go: handleConn)

  1. Handler(): path /derp/probe hoặc /relay/probe → 200 OK (health)
  2. Yêu cầu header Upgrade: DERP; thiếu → 426 Upgrade Required
  3. Hijack() lấy TCP thô; gửi 101 (trừ khi DERP-Fast-Start: 1, Tailscale ≥ v1.86)
  4. FrameServerKey → đọc FrameClientInfo (deadline 10s) → OpenFrom xác thực → FrameServerInfo
  5. Đăng ký clientConn vào clients[clientKey]
  6. defer: rớt → xoá clients + sessions.RemoveAll(clientKey)

F2 — Định tuyến gói (dispatch → route) tính năng cốt lõi

FrameSendPacket payload = destPubKey[32] + wgData. route() xét cây 3 nhánh cho mỗi gói:

destKey đang kết nối TCP ở relay này?
Có — Option 1 (local TCP): destCC.sendRecvPacket(srcKey, wgData) = FrameRecvPacket trên TCP của dest. Zero-hop, nhanh nhất (DERP cổ điển).
Không → EndpointCache.Resolve có UDP endpoint?
Có — Option 2 (cầu UDP): sessions.GetOrCreate(src,dst) → SendUDP(wgData) tới dstAddr. Reply UDP → onReply → src.sendRecvPacket(dstKey, reply) (bơm vào luồng DERP của src). Send lỗi → Invalidate(dst) + Remove(session).
Không — Option 3: DROP + log “no path” (bình thường không xảy ra).
src → relay
FrameSendPacket(destKey, wgData)
relay → EndpointCache
Resolve(destKey) → udpAddr (CurAddr / public Addr)
relay → Session
GetOrCreate(src,dst,udpAddr,onReply) → SendUDP(wgData)
Session → dst
gói WireGuard qua UDP → dst trả reply UDP
Session → relay → src
onReply(dstKey, reply) → FrameRecvPacket(src=dstKey, reply) lên TCP của src

F3 — Khám phá endpoint (endpoint.go: EndpointCache)

  • Run(ctx): nền, mỗi cache-ttl (mặc định 5s) gọi refresh đọc LocalClient.Status() của sidecar.
  • Phân loại: CurAddr != "" && Relay == "" → UDP-capable; ngược lại TCP-only.
  • Resolve(destKey) ưu tiên (1) CurAddr, (2) endpoint public đầu tiên trong Addrs (thử lạc quan). Không có → TCP-only.
  • Invalidate(k): xoá cache 1 peer khi UDP send lỗi → lần sau refresh lại.

F4 — Quản lý session (session.go: SessionTable)

  • 1 session = cặp (src, dst) + 1 UDP socket tạm riêng (ListenUDP port 0) → reply không lẫn, nhiều session tới cùng dst dùng port nguồn khác nhau.
  • GetOrCreate: có sẵn thì trả lại (cập nhật dstAddr nếu roaming); chưa có thì tạo socket + chạy readLoop.
  • readLoop: đọc UDP từ dst; lọc theo IP nguồn (đúng dst IP; cho phép đổi port do NAT roaming) → onReply(dstKey, payload).
  • Remove / RemoveAll: đóng socket khi xoá 1 session / khi client src rớt.

F5 — Keepalive & Ping/Pong (server.go)

  • Vòng chính dùng kaTicker 60s → FrameKeepAlive. Đọc frame trong goroutine riêng để keepalive vẫn tick khi không có dữ liệu.
  • framePing (8 byte) → echo framePong. frameNotePreferred → bỏ qua. Frame lạ → bỏ qua (forward-compat).

F6 — Lưu server key bền (main.go: loadOrGenKey)

  • Đọc key từ -key-file (/data/relay.key, volume relay_data). Hỏng/không có → sinh key.NewNode() rồi ghi (0600).
  • Vì sao quan trọng: serverPubKey trong FrameServerKey phải ổn định qua restart, nếu không client thấy “server lạ”.

F7 — Health probe & graceful shutdown

  • GET /derp/probe / /relay/probe → 200 OK (CI/CD & load balancer; deploy vpn6 kiểm …/relay/probe).
  • SIGINT/SIGTERM → httpSrv.Shutdown(5s) + huỷ context (dừng cache refresh).

4 · Di chuyển sang server mới (server migration)

4.1 Chuyển một relay (vpn5/vpn6) sang VPS mới

  1. DNS vpnX.hangocthanh.io.vn trỏ IP VPS mới; mở ufw 41641/udp.
  2. Bảo toàn relay_data (chứa relay.key) nếu muốn giữ server pubkey → client không học lại key. Không copy thì relay sinh key mới (client cập nhật qua DERPMap).
  3. Deploy: docker compose up -d --build trong relay-vpnX/. Sidecar join lại bằng TS_AUTHKEY; proxy xin cert mới.
  4. Cập nhật DERPMap (derp.urls / dashboard derp-backend) nếu IP/region đổi.
  5. Xác minh: curl -k https://vpnX.../relay/probe = OK; log thấy “endpoint cache refreshed: N peers”.

4.2 Lưu ý transport

  • Proxy phải giữ Upgrade/WebSocket và không đặt timeout ngắn cho kết nối DERP (dài hạn — ReadTimeout/WriteTimeout = 0).
  • Đổi proxy (Traefik ↔ Caddy): chỉ cần route Host → relay:8080 + passthrough header; giao thức DERP không đổi.
