DERP thuần — derper chính chủ

DERP relay chính chủ của Tailscale (derp-vpn3/, derp-vpn4/): nó là gì · giao thức · call flow từng tính năng — và khác gì với DERP embed.  ·  Bản đọc offline; nguồn chuẩn là DERP-PURE.md.

0 · DERP thuần là gì?

Đây là derper chính chủ của Tailscale (reference implementation), build từ source tailscale.com/cmd/derper pin v1.80.0. Chạy độc lập, tự quản TLS, làm 2 việc cốt lõi:

Vai tròCổngÝ nghĩa
DERP relay:443 HTTPS/WebSocketChuyển tiếp gói WireGuard giữa client khi không đi direct được (fallback)
STUN server:3478/udpGiúp client phát hiện địa chỉ public để thử direct (đục NAT)
ACME HTTP-01:80Let's Encrypt xác thực domain để cấp cert
Triết lý: trong Tailscale, direct (WireGuard UDP P2P) là ưu tiên, DERP là dự phòng. STUN của derper giúp client đi direct; chỉ khi không đục được NAT mới chạy qua DERP. (DERP embed mới là bản đặc biệt cho mạng chặn UDP — xem §5.)
Bảo mật: derper không đọc được nội dung — payload là WireGuard mã hoá đầu-cuối; derper chỉ thấy public key để định tuyến. --verify-clients=false = relay mở (dữ liệu vẫn an toàn); có thể siết theo tailnet (F4).

1 · Giao thức

DERP thuần nói đúng giao thức DERP v2 — vì nó chính là bản tham chiếu mà DERP embed mô phỏng. Khung tin (frame 5-byte header + payload), bảng frame types, magic DERP🔑, handshake 5 bước giống hệt DERP-EMBED §1 — không lặp lại. Khác biệt nằm ở transport & tính năng:

2 · Call flow từng tính năng

F1 — Relay gói giữa client (chức năng DERP cốt lõi)

derper là một switchboard mã hoá: client gửi gói kèm public key đích, derper chuyển tới client đích đang nối.

A / B → derper
Mỗi client: Upgrade DERP qua TLS 443 → handshake (ServerKey/ClientInfo/ServerInfo)
A → derper
FrameSendPacket(destKey=B, wgData)
derper → B
FrameRecvPacket(srcKey=A, wgData) — chuyển theo public key, KHÔNG giải mã được
derper
Khi 1 peer rớt → gửi FramePeerGone(peerKey) cho client liên quan; FrameKeepAlive định kỳ
Nhiều client cùng 1 derper = mesh sao. derper còn hỗ trợ mesh giữa nhiều DERP server (mesh key) — triển khai này chạy 1 node/region, không bật mesh.

F2 — STUN server (--stun :3478) — bệ phóng cho DIRECT

Tính năng giúp tránh phải dùng DERP. Client cần biết IP:port public của mình (sau NAT) để bên kia gửi UDP trực tiếp.

Client → STUN :3478
STUN Binding Request (UDP)
STUN → Client
Binding Response = địa chỉ public (IP:port sau NAT)
Client
netcheck dùng kết quả + đo RTT từng DERP region → chọn preferred DERP
Client → Peer
thử gửi WireGuard UDP trực tiếp (hole punching)
Đục NAT thành công → DIRECT (P2P), KHÔNG qua DERP
Thất bại → fallback relay qua DERP (F1)

STUN là một phần của netcheck client chạy để chọn DERP gần nhất và hole-punch → 1 DERP tốt vừa relay vừa chạy STUN.

F3 — TLS / cấp chứng chỉ (--certmode=letsencrypt)

Khác DERP embed (proxy lo TLS), derper tự xin và tự phục vụ cert.

derper → Let's Encrypt
yêu cầu cert cho --hostname (PHẢI khớp DNS → IP VPS)
LE → cổng 80
thử thách HTTP-01 tại /.well-known/acme-challenge/…
derper (cổng 80) → LE
trả lời đúng token → LE cấp cert
derper
lưu cert vào --certdir=/data/derper-certs (volume) → phục vụ HTTPS :443, tự gia hạn
Cần: DNS --hostname trỏ đúng IP VPS, cổng 80 và 443 mở ra Internet. Cert ở volume derper_certs → sống qua restart.

F4 — Kiểm soát truy cập (--verify-clients)

Giá trịHành viKhi nào dùng

vpn3/vpn4 hiện đặt false. Bật true là thay đổi hành vi → cần plan/duyệt riêng.

F5 — Đăng ký vào headscale DERPMap (client học được relay này)

  • headscale (vpn2) phục vụ DERPMap (vd derp.urls / derpmap.json); mỗi region khai hostname, port, có STUN hay không.
  • Client tải DERPMap → netcheck (F2) đo RTT từng region → chọn preferred DERP → khi cần fallback thì nối relay đó (F1).
  • Region code (vd vpn3-vn, vpn4-vn) là nhãn trên dashboard, khớp REPORTER_NAME của ping-reporter.

F6 — Quan hệ với sidecar & ping-reporter (giám sát)

Quan trọng: trong derp-vpn3//derp-vpn4/ có container tailscale (sidecar) và ping-reporter, nhưng derper KHÔNG cần sidecar để relay. derper là relay công khai, độc lập. Sidecar chỉ để: join tailnet làm node vpn3/vpn4 (hiện trên dashboard) và cho ping-reporter đo latency (PING-REPORTER). Sidecar chết → relay vẫn chạy, chỉ mất số liệu giám sát. (Trái lại, DERP embed cần sidecar để biết UDP endpoint.)

3 · Triển khai

Tham số derper (vpn3 — vpn4 chỉ khác hostname)

--hostname=vpn3.hangocthanh.io.vn   # PHẢI khớp DNS → IP VPS
--a=:443                            # HTTPS / DERP WebSocket
--http-port=80                     # ACME HTTP-01 + HTTP
--certmode=letsencrypt             # derper tự quản TLS
--certdir=/data/derper-certs       # volume derper_certs (giữ cert qua restart)
--stun                             # bật STUN server
--stun-port=3478                   # cổng STUN (UDP)
--verify-clients=false             # relay mở (xem F4)

Cổng & build

  • Cổng ra Internet: 80/tcp (ACME), 443/tcp (HTTPS+DERP), 3478/udp (STUN).
  • Build: golang:1.23-alpine → go install tailscale.com/cmd/derper@v1.80.0 → runtime alpine:3.20. Pin version để tái tạo image.
  • Không secret: DERP thuần không cần key (.env.example chỉ tham khảo DERP_DOMAIN). vpn4 dùng lại Dockerfile.derper của vpn3.

Di chuyển sang server mới (server migration)

  1. DNS vpnN… trỏ IP VPS mới; mở 80/tcp, 443/tcp, 3478/udp.
  2. docker compose up -d --build trong derp-vpnN/. derper tự xin cert mới qua ACME (cổng 80 reachable). Không bắt buộc copy volume cert.
  3. Cập nhật DERPMap trên headscale nếu IP/region đổi. DNS giữ hostname → client tự kết nối lại sau khi DNS trỏ IP mới.
  4. (Tùy chọn) copy volume tailscale_vpnN để sidecar giữ identity node giám sát.
  5. Xác minh: curl -I https://vpnN… (cert hợp lệ); tailscale netcheck thấy region; dashboard có latency.

5 · DERP thuần vs DERP embed — chọn cái nào?

Tiêu chíDERP thuần (derper)DERP embed (hybrid)
Bản chấtrelay chính chủ + STUNrelay tự viết + cầu UDP
Mạng chặn UDP (Squid)?client vào được qua 443, nhưng peer-to-peer vẫn fallback DERP 2 chặngtối ưu: relay tự đi UDP tới peer → gần direct
STUN/NAT traversalcó — giúp client đi directkhông (giả định mạng đã chặn UDP)
TLStự quảnsau reverse proxy
Độ phức tạp vận hànhthấp (1 binary chính chủ)cao hơn (code riêng + proxy)
Khi nào dùngDERP region phổ thông cho cả tailnetđiểm vào cho mạng chặn UDP cứng
Tóm lại: derper là xương sống DERP tiêu chuẩn (mọi node hưởng lợi từ STUN + fallback). embed là giải pháp đặc thù đặt ở rìa mạng bị chặn UDP. Hai cái bổ sung, không thay thế nhau.
